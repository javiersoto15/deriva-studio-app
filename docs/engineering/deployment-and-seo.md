# Deployment + SEO/Discovery

How the site ships to production, and how it surfaces in Google Search and Google Maps.

## Production deploys

- **Host**: Vercel (project linked via `.vercel/project.json`)
- **Production alias**: `https://derivastudio.cl`
- **Deploy**: `vercel deploy --prod` (or `vercel deploy` for preview)
- **Framework**: Next.js 16 App Router on Fluid Compute
- **CLI**: Vercel CLI 50.32.5 (consider upgrading; current latest is 53.x)

## Standing rules

1. **Paper-first**: every visual feature must be mocked on Paper artboards (mobile + desktop) before deploy. The user reviews on the canvas before any ship.
2. **Explicit deploy approval**: even after a prior deploy approval, the next deploy requires fresh confirmation in the chat. Do not chain deploys.
3. **Don't skip hooks**: never `--no-verify` or `--no-gpg-sign`.

## Domain + DNS

The `derivastudio.cl` domain is connected to Vercel and resolves via Vercel's edge network. DKIM / SPF / DMARC records for the `hola@derivastudio.cl` sender are configured in Resend's domain verification panel.

## SEO surfaces ‚Äî what Google looks at

### 1. Favicon next to the URL in search results

Google pulls the favicon from several sources, in roughly this order:

- `/favicon.ico` at the root ‚Äî **served from `app/favicon.ico`**, multi-resolution (16/32/48px), generated from `public/brand/isotipo-verde.svg` via Sharp + `png-to-ico`
- `<link rel="icon" href="/icon.svg">` ‚Äî Next.js App Router convention, served from `app/icon.svg`
- `<link rel="apple-touch-icon" href="/apple-icon.png">` ‚Äî served from `app/apple-icon.png`
- JSON-LD `Organization.logo` field ‚Äî Knowledge Panel signal

All four are configured. Favicon updates on Google's SERP can take **1‚Äì4 weeks** to propagate after a fix.

### 2. JSON-LD structured data

`app/page.tsx` injects three linked schema.org blocks via `<script type="application/ld+json">`:

- `Organization` (`@id`: `${siteUrl}/#organization`) ‚Äî has `logo: { @type: ImageObject, url, width, height, contentUrl }`, `sameAs` linking to Instagram
- `CafeOrCoffeeShop` (`@id`: `${siteUrl}/#cafe`) ‚Äî local-business details, opening hours, geo coordinates, `parentOrganization: { @id: ".../#organization" }`
- `WebSite` (`@id`: `${siteUrl}/#website`) ‚Äî publisher reference to the Organization

The cross-references via `@id` let Google's crawler treat them as a single linked graph.

**Validate with:**
- https://search.google.com/test/rich-results
- https://validator.schema.org/

### 3. Google Maps listing

**Google Maps and the Knowledge Panel pull the logo from Google Business Profile (GBP), NOT from the website.** There is no website-side fix for the Maps logo. To set it:

1. Go to https://business.google.com/create
2. Search for `Magnere 1570 Local 105, Providencia, Santiago` ‚Äî claim the auto-created listing if one exists, or create new
3. Verify ownership (postcard typical in Chile, 5‚Äì14 days)
4. Upload logo (square ‚â• 250√ó250, ideally 720√ó720) ‚Äî use `public/brand/isotipo-verde@3x.png` (1080√ó1080)
5. Set Website field to `https://derivastudio.cl` to consolidate signals
6. Logo appears in Maps + Knowledge Panel within 24‚Äì72h after verification

This step is **user-side only** and cannot be automated from this codebase.

### 4. Pushing Google to re-crawl

After SEO changes, request indexing:

- Search Console ‚Üí URL Inspection ‚Üí enter `https://derivastudio.cl` ‚Üí Request Indexing

This pushes Google to look again. Favicon SERP update timing is still subject to Google's own schedule.

## OG share card

`app/opengraph-image.tsx` uses `next/og` (`ImageResponse`) to generate the 1200√ó630 social card at build time. It avoids Google Fonts fetches by rendering diamond glyphs as rotated `<div>` shapes (the `‚óÜ` character requires a font that doesn't load reliably during the static build).

## Asset variants on `public/brand/`

| File | Purpose |
|---|---|
| `isotipo-verde.svg` | Source vector ‚Äî green isotipo |
| `isotipo-verde@1x.png` / `@2x` / `@3x` | Raster for emails + Google logos (light mode) |
| `isotipo-sage@1x.png` / `@2x` / `@3x` | Raster for emails (dark mode) ‚Äî generated from SVG with `#8FAE8C` fill |
| `logo-con-isotipo.svg` + 1x/2x/3x | Full lockup with wordmark |
| `wordmark.svg` | Text-only wordmark |
| `CrownAvenue-Regular.otf` | Brand display font (not used in webapp yet) |

The sage variants are generated by a one-off Sharp script (see git history for the generation snippet). Re-run only if the SVG source changes.
